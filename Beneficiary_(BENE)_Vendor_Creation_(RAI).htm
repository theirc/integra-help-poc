Business Process Overview

Beneficiaries are created as a type of Vendor record in Integra. Use the steps provided as a reference when creating or updating BENE records in Integra.

Vendor Record Creation

In Integra, the details of all persons and organizations with whom the IRC deals are stored in the global address book.

  • The BENE Vendors group relates to any person who is targeted for assistance, who directly benefits from, or comes in contact with, an IRC-implemented or financially supported IRC program.

Most clients are automatically created in Integra through the ETO interface; however, non-ETO clients who do not need heightened confidentiality (i.e., their name being saved in Integra is acceptable) can be manually entered and approved by the local Requestor. Examples of a Requestor include but are not limited to the following roles:

  • Caseworker
  • Casework Supervisor
  • Resettlement Manager
  • Anti-Trafficking Manager
  • Finance Manager

Clients pulled in from ETO come over with active status. However, clients created directly in Integra must be submitted to workflow approval or the status will be on hold. (Note: When submitting the client to workflow approval, the system auto-approves the record.)

If the office has a program(s) where clients require heightened confidentiality, Finance will create a generic vendor record in the Vendor module. This is done during the project set-up phase. Example of a such a vendor includes anti-trafficking clients.

Due to the sensitivity of this vendor type and to ensure the record is secured to the legal entity, only those with permission can view the record.

Create Beneficiary (standard confidentiality)

Purpose

Use this procedure to create a beneficiary vendor in Integra with standard confidentiality needs.

  • BENE = Beneficiary (Client)
  • Requestor (Caseworker, Casework Supervisor, Resettlement Manager, Anti-Trafficking Manager)

Prerequisites

  • The vendor does not already exist in Integra
  • The client is a non-ETO client
  • It is confirmed that the client does not require heightened confidentiality

Steps

  1. Navigate to Common > Common > Global address book.

  1. On the menu at the top of page, click New.

  1. Under the General tab, complete the following fields:

Field

Description and/or Action Required

Party ID

This value is automatically assigned. No entry required.

Type

Select Person from the drop-down selection list.

First Name

Enter the first name of the vendor.

Last name

Enter the last name of the vendor.

Gender

Select Male or Female from the drop-down selection list.

Address books

Select BEN (USP Beneficiaries) from the drop-down selection list.
  1. Scroll down the form and open the Addresses section.
Note: Since payments are often made in person to a beneficiary, address information in the BENE vendor record may be optional.

  1. Click Add.

  1. Complete the following fields:

Field

Description and/or Action Required

Name or description

Enter the name or the description you are assigning to the address.

Purpose

Select one of the of location options from the drop-down list.

ZIP/postal code

Enter the applicable zip code.

Street

Enter the street address.

Street number

Enter the street number.

City

Enter the city.

State

Enter the state.

Primary

Select Yes if the address is the “primary” location.
  1. Click OK to proceed.

  1. To verify if this entry already exists, scroll up to the top of the page (upper left), click Check for duplicates.

  1. Select the values to include in the duplicate check, e.g., First name, last name.
  2. Click Search.
  • Any duplicate records will be listed in the bottom section of panel. If a duplicate record is found, cancel the creation of the new record.
  1. If there are no duplicates found, click OK to proceed.

  1. Click Save on the menu at the top of the page.
  2. Under the New fast tab, click Vendor to access the New Record page.

  1. In the Group field, select BENE from the drop-down selection list.
  2. Scroll down and open the Purchasing demographics section.

  1. In the Currency/Donor Currency field, select the currency the vendor wishes to be paid in.
  2. Scroll down and open the Payment section.

  1. In the Method of payment field, select the expected payment method
    • Cash
    • Check
    • Prepaid Card
    • Bank Transfer
    • EFT_Chase
  2. Select the method of payment.
Important Reminder: If the payment method type is anything other than cash, bank account information including bank name is required. 

Complete the following steps to enter banking information:

  • Click Bank accounts (top of page)
  • Click New on the set-up page
  • Enter a Bank account description and the Name of the bank
  • Enter the Routing number and the Bank account number
  • Enter the IBAN number – if applicable
  1. Click Save on the menu at the top of the page.
  2. Click X (top right) to close the page
Note: The newly created vendor is placed on hold automatically.
  1. In the action pane, click DynAd Workflow then Submit to send for approval.

  1. In the Vendor workflow - Submit form, enter a comment in the Comment field and click the Submit button.

Result

The beneficiary vendor is created and is automatically approved.

Create Beneficiary (with heightened confidentiality)

Purpose

Use this procedure to create a beneficiary vendor record in Integra when there are heightened confidentiality requirements.

BENE = Beneficiary (Client)

This is a generic record to be used to pay a range of clients and should not reflect any one client’s information except for the moment it is being used to make a payment. Finance is responsible for editing the generic vendor record’s address and subsequently removing any identifying information once the payment is complete.

Important: Only one (1) generic vendor record is created for each office and this record is continuously used for all confidential clients.

Responsibilities

  • Requestor (Caseworker, Casework Supervisor, Resettlement Manager, Anti-Trafficking Manager)
    • Edits the generic vendor record and submits payment request (invoice)
    • Approver group (Casework Supervisor) receives, reviews, and approves invoice and sends to Finance
  • Finance Manager (Accounts Payable)
    • Follows process to review and post vendor payment journal

Prerequisites

  • The vendor does not already exist in Integra
  • The client is a non-ETO Client
  • It is confirmed that the client requires heightened confidentiality

Steps

  1. Navigate modules to Common > Common > Global address book.
  2. Click New on the Action Pane at the top of the Party page.

  1. Under the General tab, complete the following fields:

Field

Description and/or Action Required

Party ID

Auto generated.

Type

Select Person.

First Name

Type the word GENERIC.

Last name

Type the word CLIENT.

Gender

Select Male or Female or Non-Specific.

Address books

Select BEN.

  1. Click Vendor at the top of the page to open the vendors New Record page.

  1. In the New Record page, enter or select a value in the Group field to specify the type of vendor. Select BENE for Beneficiary (Client). Click Save.

  1. Scroll down to the Payment tab, select the expected payment method in the Method of payment field. Click Save again.
  2. Select the applicable method of payment:
    • Cash
    • Check
    • Prepaid Card

  1. In the action pane, click DynAd Workflow then Submit to send for Approval.

  1. In the Vendor workflow - Submit form, you may enter a comment in the Comment field and click Submit.

Result

A generic beneficiary vendor has been created and will be automatically approved.

Next Steps

  1. The requestor submits a payment request in Integra
  2. Finance reviews and posts the payment request invoice and processes payment.

Create a Client ID Financial Dimension for Confidential Client

Purpose

Finance uses this procedure to create a Client ID financial dimension (D9) in Integra for the confidential client using the number supplied by the requestor.

This D9 value is managed by the program staff responsible for supporting the client and this record should not include anything other than the value provided.

Prerequisites

  • Requestor has provided financial dimension number

Steps

  1. Navigate modules to General ledger > Chart of accounts > Dimensions > Financial dimensions.
  2. Scroll down to click on the financial dimension ClientID (D9) then click Dimension values.

  1. Click New in the Action pane to create a new D9.

  1. Type a value in the Dimension value field and copy that value into the Description field. Click Save.

Result

The client ID financial dimension has been created for a generic client with confidential needs.
