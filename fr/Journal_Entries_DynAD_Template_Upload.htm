Le modèle DynAd

Le modèle DynAd est un outil permettant de télécharger des écritures de journal en masse dans Integra.

Le Modèle Dynad est particulièrement utile lorsque le département Finances doit diffuser des écritures en masse depuis le grand livre d’une entité juridique d’Integra à une autre. Il facilite aussi les diffusions entre différents modules (livres auxiliaires), et entre le grand livre et des modules, au sein de la même entité juridique.

  • Vers/depuis des livres auxiliaires/modules différents
  • Vers/depuis un grand livre
  • Vers le grand livre depuis des livres auxiliaires/modules
  • Vers des livres auxiliaires/modules depuis le grand livre

Règle générale : utilisez le Modèle DynAd s’il n’y a pas plus de 10 Écritures de journal à télécharger ; sinon, créez des écritures de journal directement dans Integra.

Il existe deux modèles différents pour s’adapter aux différentes devises :

  • Modèle de journal DynAd (utilisé pour les transactions en USD/devises de base)
  • Autre montant dans le modèle de journal DynAd (utilisé pour toutes les transactions dans des devises autres que l’USD)

La différence entre les deux modèles ci-dessus est qu’il y a deux colonnes supplémentaires dans la version non USD/de base pour prendre en compte le type de taux de change et le taux de change utilisé.

Traitement des écritures de journal en devises étrangères

Lors de l’importation depuis le Modèle DynAd :

  • la date de transaction détermine quel taux de change utiliser, en cas de reclassement d’une écriture depuis/vers la période actuelle.
  • Le montant dans la devise de transaction est converti à l’aide du taux d’Integra et diffusé au champ du montant dans la devise de base/comptabilité.
  • Vérifiez que le taux de change est saisi en tant que facteur de 100. Par exemple, un taux de change saisi de 1,1463 sera converti en 114,63 dans le journal de téléchargement.

Par ailleurs, la compensation monétaire entre des devises peut également être effectuée à l’aide du journal DynAd. Cela est souvent le cas pour les comptes d’entrées de fonds envoyés depuis le siège social mais reçus dans la devise locale.

Créer une écriture de journal dans Integra

Objectif

Utilisez cette procédure pour créer une écriture de journal directement dans Integra.

Cette procédure permet de diffuser des écritures de journal relatives à la paie, intersociétés, aux charges à payer, aux allocations de paiements préalables, aux avances, aux allocations de coûts de programme partagé, aux fournisseurs, aux clients et aux écritures de reclassement.

Conditions préalables

  • Le type de journal est connu (consultez le document Types de journaux dans Integra pour obtenir une liste complète de types de journaux et d’utilisations)
  • Les aspects financiers sont connus

Étapes

  1. Accédez aux modules Grand livre > Saisies du journal > Journaux généraux.

  1. Dans le menu en haut, cliquez sur Nouveau.

  1. Une ligne est ajoutée au haut du tableau. Dans le champ Nom, cliquez sur la flèche du menu déroulant et sélectionnez le type de journal concerné dans la liste de sélection, p. ex., JE (pièce de journal) ; GL_IC (Transaction intersociétés), etc.

  1. Dans le champ Description, remplacez le texte par défaut par une description significative du journal.

  1. Dans le menu en haut, cliquez sur Lignes.

  1. Vérifiez qu’une nouvelle ligne de journal a été ajoutée au tableau. (Sinon, cliquez sur Nouveau juste au-dessus du tableau pour ajouter une ligne.)
  2. Renseignez/validez les champs suivants sur la ligne de la pièce du journal :

Champ

Saisie requise

Date

Indique par défaut la date du jour mais peut être remplacé.

Pièce

Il s’agit d’un numéro attribué par le système qui ne peut pas être modifié. 

Société

Cette valeur indique par défaut l’entité juridique à laquelle vous êtes connecté mais peut être remplacée.

Si l’écriture concerne votre entité juridique actuelle, ne modifiez pas le code de la société.

Type de compte

Si l’écriture de journal est une écriture de livre, laissez valeur par défaut Livre.

Si l’écriture implique d’autres livres auxiliaires/modules tels que le projet, le client, le fournisseur, la banque ou les immobilisations, sélectionnez le livre auxiliaire/module concerné dans la liste de sélection du menu déroulant.

Compte

La liste de sélection du menu déroulant dans ce champ dépend du Type de compte déjà sélectionné.

  • Si le type de compte Livre a été sélectionné, la liste de sélection du menu déroulant affiche le compte principal et les champs financiers.
  • Si le type de compte Client a été sélectionné, la liste de sélection du menu déroulant affiche les comptes client.
  • Si le type de compte Fournisseur a été sélectionné, la liste de sélection du menu déroulant affiche les comptes fournisseur.
  • Si le type de compte Projet a été sélectionné, la liste de sélection du menu déroulant affiche les ID de projet.
  • Si le type de compte Banque a été sélectionné, la liste de sélection du menu déroulant affiche une liste des comptes bancaires pour l’entité juridique.

Description

Saisissez une description significative pour la ligne de la pièce du journal.

Débit

Le contenu des champs Débit/Crédit dépend du type de compte sélectionné et de la nature de la transaction.

Crédit

Le contenu des champs Débit/Crédit dépend du type de compte sélectionné et de la nature de la transaction.

Catégorie

Ce champ s’affiche uniquement si Projet a été sélectionné dans le champ Type de compte.

Dans le champ Catégorie, sélectionnez la catégorie correspondant à votre transaction. L’ID de projet (D1) saisi dans le champ Comptedétermine les catégories disponibles dans la liste, c.-à-d., vous verrez uniquement les catégories correspondant à cet ID de projet.

Devise

Laissez la valeur par défaut sur USD si vous réalisez des transactions en dollars ; sinon, mettez à jour la devise adéquate.

Taux de change

Le champ du taux de change indique automatiquement par défaut le taux de change de période de système de la devise que vous souhaitez utiliser.

Note : pour les écritures de reclassement de devises, veillez à saisir le même taux de change que celui associé à la transaction initiale réalisée pendant la période précédente. Si vous diffusez sans le taux de change initial et laissez le champ vide, le taux de change est appliqué automatiquement dans le système.

 

  1. Dans le menu affiché juste au-dessus du tableau, cliquez sur Aspects financiers et sélectionnez Compte dans la liste de sélection du menu déroulant.

  1. Renseignez les autres champs d’aspects requis en fonction de la structure de comptable de l’IRC. Par exemple, une transaction de projet requiert un code D1-D2-D3-D4-D5-D6. Le code D7 est uniquement requis pour le salaire et les avantages sociaux.
Note : un seul compte est utilisé pour les filiales. Toutes les filiales ayant des charges doivent être listées 786500.0.e pour la catégorie de projet D2.
  1. Fermez la fenêtre des Aspects financiers pour revenir à l’écran des lignes de pièce.

  1. Pour ajouter la ligne de compensation, cliquez sur Nouveau juste au-dessus de la grille du tableau.

  1. Renseignez tous les champs requis dans la ligne de pièce de la ligne de compensation.
  2. Pour saisir les aspects financiers de la ligne de compensation, cliquez sur Aspects financiers et sélectionnez Compte de compensation.
  3. Renseignez les aspects financiers de la ou des lignes de compensation.

  1. Dans le menu en haut de la page, cliquez sur Valider et sélectionnez Simuler la diffusion.
  2. Une fois la simulation de diffusion confirmée, fermez la page de la pièce du journal pour revenir à la page Journaux généraux.
  3. Dans la liste des journaux, sélectionnez le journal qui vient d’être créé et joignez les documents à l’appui. (Cliquez sur l’icône du trombone en haut à droite)
  4. Enfin, soumettez le journal au Flux de travail en cliquant sur Flux de travail sur le menu en haut de la page.

Résultat

Vous avez créé une écriture de journal dans Integra.

Créer un journal périodique

Objectif

Utilisez cette procédure pour créer des journaux qui nécessiteront uniquement quelques modifications afin de les utiliser au cours de la prochaine période comptable. Des exemples fréquents de journaux périodiques sont les allocations et les dépenses payées d’avance.

Note : pour les journaux périodiques contenant plus de dix lignes, utilisez le Modèle DynAd pour télécharger des lignes de journal. Consultez les instructions mentionnées dans l’article Modèle DynAd de cette section.

Conditions préalables

  • Les aspects financiers sont connus.

Étapes

  1. Accédez aux modules Grand livre > Écritures du journal > Journaux généraux.

  1. Cliquez sur Nouveau et sélectionnez PER_JE dans le champ Nom.
  2. Dans le menu en haut, cliquez sur Lignes.

  1. Dans le champ Type de compte, sélectionnez l’option pertinente dans la liste de sélection du menu déroulant.
  2. Dans le champ Compte, renseignez le champ avec l’écriture pertinente.
  3. Saisissez une description dans le champ Description.
  4. Saisissez le montant dans le champ Débit.
  5. Cliquez sur l’onglet Aspects financiers.

  1. Renseignez les aspects financiers concernés pour la ligne et cliquez sur OK.
  2. Sauvegardez le journal périodique.

Résultat

Vous avez créé un journal périodique.

Créer une saisie dans le journal à partir d’un journal périodique

Pour accéder au journal périodique déjà créé, suivez les étapes ci-dessous :

  1. Accédez à Grand livre > Écritures du journal > Journaux généraux

  1. Cliquez sur Nouveau.
  2. Cliquez sur Lignes dans la barre d’outils en haut.

  1. Dans le menu en haut de la page, cliquez sur Journal périodique et sélectionnez Récupérer le journal.
  2. Dans la fenêtre contextuelle, sous le numéro de journal périodique, sélectionnez le numéro de journal périodique que vous avez déjà créé.

  1. Dans le champ Jusqu’à présent, saisissez une date. Il s’agit de la date de création du journal.

  1. Dans le champ de journal Copier ou déplacer, sélectionnez Copier.
  2. Cliquez sur OK.
  3. Au besoin, apportez les modifications requises.
  4. Mettez à jour tous les autres champs requis dans les lignes de pièce, p. ex., n° de facture, aspects financiers, etc.
  5. Cliquez sur Sauvegarder.
  6. Soumettez le journal à l’approbation du Flux de travail.

Résultat

Vous avez créé une écriture de journal à partir d’un journal périodique. Une fois approuvé, le département Finances diffuse le journal.

Ajuster la comptabilité de la transaction de projet diffusée (Méthode facile)

Utilisez cette procédure pour corriger les erreurs concernant les codes ou aspects financiers du compte principal.

Cette procédure conserve la diffusion initiale et en crée de nouvelles.

Étapes

  1. Accédez à Gestion de projet et comptabilité > Projets > Tous les projets.

  1. Recherchez et sélectionnez la ligne de projet pertinente.
  2. Dans le menu en haut de la page, cliquez sur Gérer et sélectionnez Transactions diffusées.

  1. Sélectionnez la transaction de projet diffusée à ajuster et cliquez sur Ajuster la comptabilité sous l’onglet Processus.

  1. Une boîte de dialogue s’ouvre et affiche les écritures comptables actuelles dans la section supérieure de la boîte de dialogue. Examinez les écritures actuelles.
  2. Dans la section inférieure, saisissez un nouveau compte principal et/ou les aspects financiers pour recoder la transaction, selon les besoins.
  3. Faites défiler jusqu’en bas de la boîte de dialogue.

  1. Cliquez sur Aperçu pour vérifier que toutes les valeurs sont correctes, et cliquez sur Diffuser pour diffuser la transaction recodée.

Résultat

Vous avez ajusté la comptabilité d’une transaction de projet diffusée.

Recoder une saisie dans le journal

Le recodage permet généralement de corriger les erreurs concernant les codes ou aspects financiers du compte principal.

Cette tâche est réalisée par le personnel du département Finances tous les jours ou tous les mois.

Voici les différentes manières d’effectuer un recodage :

Créer un journal d’ajustement de projet

  • Cela consiste à ajuster les transactions de projet dans le livre auxiliaire du projet, ce qui ajustera automatiquement la diffusion du grand livre correspondant.
  • Vous pouvez créer des journaux d’ajustement de projet à l’aide du modèle de journal DynAd pour les téléchargements en masse entre le grand livre et le livre auxiliaire du projet.
  • Vous pouvez ajuster les transactions en fonction du projet.
  • Vous pouvez ajuster en fonction du lot.

Saisir les corrections de bilan

  • Cela permet principalement de corriger les codes de compte erronés à partir du grand livre.

Étapes

Étapes d’ajustement des transactions de projet par projet dans Integra :

  1. Accédez à Gestion de projet et comptabilité > Projets > Tous les projets.
  2. Cliquez sur ID de projet et dans le menu déroulant, recherchez l’ID de projet à ajuster (sélectionnez « contient » pour chaque recherche facile).

  1. Dans Gérer > Processus, cliquez sur Ajuster les transactions.

  1. Dans le menu en haut, cliquez sur Sélectionner. Une fenêtre contextuelle s’affiche.
  2. Dans la fenêtre contextuelle, assurez-vous que les dépenses, les articles et/ou les frais sont configurés sur Oui en fonction du type de transaction que vous prévoyez d’ajuster.
  3. Dans Date de projet, sélectionnez ou saisissez Date de début et Date de fin. Il s’agit de la période pour laquelle vous voulez extraire votre transaction à ajuster.
  4. Dans Date du livre, sélectionnez ou saisissez la date. Il s’agit de la période au cours de laquelle vous voulez diffuser l’ajustement.
  5. Faites défiler jusqu'à l’en-tête Enregistrements à inclure. La fonction Enregistrements à inclure vous permet de sélectionner plusieurs ID de projet à ajuster en même temps. À ce stade, vous n’utiliserez pas cette fonction.
  6. Cliquez sur OK.

  1. Dans la liste de transactions, sélectionnez celles à ajuster.

  1. Cliquez sur Ajuster.
  2. Dans la fenêtre contextuelle, dans ID de projet, saisissez l’ID de projet ou le code D1 vers lequel vous souhaitez déplacer la transaction.
  3. Dans le champ Catégorie, sélectionnez la catégorie de projet adéquate ou le code D2 code pour lequel vous souhaitez ajuster la transaction.
  4. Dans le champ Propriété de la ligne, sélectionnez Facturable ou Non facturable.
Notes :
La fonction Facturable s’applique uniquement aux projets avec des frais (ICR) et requérant une comptabilisation de revenus comme les projets de subvention, de contrats ou restreints.

La fonction Non facturable est utilisée pour les projets ne requérant aucune comptabilisation de revenus tels que les projets non restreints et temporairement restreints.
  1. Si vous souhaitez attribuer plusieurs codes D1, insérez des lignes supplémentaires et modifiez la colonne de pourcentage avec la clé d’attribution adéquate. Sinon, laissez la valeur 100.
  2. Dans le champ Code employé, saisissez le code d’employé D7. Cela est uniquement requis si vous ajustez des transactions relatives au personnel.
  3. Dans le champ Code de motif, sélectionnez le code correspondant à votre ajustement.
  4. Dans le champ Commentaire sur le motif, vous pouvez remplacer l’écriture par défaut et saisir des commentaires supplémentaires concernant l’ajustement.
  5. Cliquez sur OK.

Un écran divisé indiquant les transactions que vous ajustez s’affiche en bas.

Vous pouvez vérifier les aspects financiers en haut des transactions que vous souhaitez ajuster. Les aspects financiers seront automatiquement renseignés sur la base des codes précédents. Vous pouvez donc modifier les aspects financiers en conséquence pour chaque ligne.

  1. Cliquez sur Diffuser dans le menu en haut de la page.
  2. Cliquez sur OK dans la fenêtre qui s’affiche.

Résultat

Vous avez recodé une écriture GL.

Annuler une écriture de grand livre (après une balance générale des comptes)

Utilisez cette procédure pour annuler une écriture de grand livre (GL) dans Integra.

Les annulations permettent d’annuler des écritures erronées dans le GL et de les coder ensuite correctement.

Étapes

  1. Après avoir exécuté la balance générale des comptes (voir ci-dessus), double-cliquez sur la transaction dans la colonne Solde de clôture à annuler.
  2. Cliquez sur Annuler la transaction.

  1. Dans l’écran Annulation de transaction, saisissez la Date de diffusion d’annulation.
  2. Cliquez sur OK.

Un message s’affiche indiquant que la transaction a été annulée.

  1. Cliquez sur Suivi annulé. Les transactions initiales et annulées s’affichent.
seules les transactions du grand livre peuvent être annulées. Les transactions diffusées depuis un livre auxiliaire ne peuvent pas être annulées dans le grand livre.
  1. Fermez la transaction en sélectionnant X en haut à droite de l’écran une fois l’examen terminé.

Résultat

La transaction du GL a été annulée.

Renseigner le modèle DynAd

Objectif

Utilisez cette procédure pour renseigner le Modèle DynAd afin de télécharger des transactions en masse dans différents modules d’Integra.

Étapes

  1. Récupérez la toute dernière version du Modèle DynAd dans Box.
  2. Lisez attentivement toutes les instructions suivantes intégrées dans le Modèle DynAd.
  • Instructions spécifiques au champ dans la section d’en-tête du modèle
  • Instructions spécifiques à la colonne pour chaque ligne d’écriture de journal
  1. Renseignez le Modèle DynAd avec les écritures de journal.

 

Important : ne modifiez aucune information dans les en-têtes de colonne du Modèle DynAd. Si des modifications sont apportées, le téléchargement dans Integra échouera.

Il est également possible de Diffuser des fichiers d’importation de journal à partir de cette grille. Sinon, le journal peut être diffusé en accédant à Grand livre > Écritures du journal > Journaux généraux.
  1. Sélectionnez la tuile à gauche intitulée Importer des transactions de journal.

  1. Activez les champs/boutons suivants :

Champ/Bouton

Saisie requise

Groupe de traitement

Exécuter en mode lot

  • La colonne Phase intermédiaire affiche le nombre d’enregistrements téléchargés avec succès.

  1. Cliquez sur Voir les données intermédiaires pour voir les détails des erreurs dans les lignes de journal.

  1. Dans le menu en haut de la page, cliquez sur Tout valider pour voir les messages d’erreur détaillés.
  2. Ajoutez la colonne Message d’erreur au tableau et déplacez-la dans un endroit adapté/visible de la grille.
  3. Examinez la grille afin de déterminer les lignes validées et les lignes comportant des erreurs.
  4. Cliquez sur Éditer pour éditer/corriger les lignes de la grille. Corrigez toutes les lignes ayant le statut Erreur dans Transfert vers la cible tant que toutes les lignes n’ont pas été corrigées.

  2. Cliquez sur Tout valider pour valider les lignes après modification.

  1. Cliquez sur OK.
  2. Examinez les données dans la grille Exécution des données cibles.

Annexe

Reclassement de base et transactions en devises étrangères

Pour reclasser des écritures entre deux projets, l’IRC utilise généralement les ajustements de projet, notamment concernant les écritures de reclassement recodant une dépense ARTICLE entre deux projets. Cet ajustement de projet a été personnalisé pour inclure des codes de motif et l’utilisateur peut retrouver la transaction de projet initiale de ces ajustements. Le Journal DynAd n’offre pas cette fonction.

Utilisez le Journal DynAd pour reclasser des montants entre deux bilans, et à des fins de comptabilité et de recodage intersociétés.

Vous pouvez également utiliser le Journal DynAd pour recoder les frais de projet DÉPENSE et FRAIS.

Si vous reclassez une transaction dans une devise étrangère au cours de la même période comptable, vous n’avez pas besoin de « forcer » le taux de change.

Reclassement des écritures de périodes antérieures (Période provisoire)

Pendant la période provisoire, le processus de reclassement des écritures des mois précédents sera conforme à ce qui suit :

  • toutes les écritures des périodes précédentes, qu’il s’agisse de transactions de compensation de reclassement ou intersociétés, sont diffusées avec le système mensuel actuel dans Integra.  
  • Le champ Date du document permet de suivre la date initiale des transactions.  Concernant les écritures provenant d’une période précédente mais qui ont été diffusées initialement ou reclassées à l’aide de la date de la période actuelle dans le système, le champ de la date du document doit être renseigné pour inclure la date de transaction initiale.
  • Le fichier SAF provenant du bureau local doit être téléchargé dans ISUN avec la date de transaction et la date du document.
  • Si la date de transaction d’écriture du fichier SAF tombe pendant le mois en cours, la date de transaction du système et la date du document seront identiques.
  • Si la date de transaction dans le fichier SAF tombe pendant une période antérieure, les écritures seront diffusées à l’aide de la date de la période en cours dans le système, et le champ de la date du document indiquera la date initiale de la transaction.

Affichage des types de journaux dans Integra

Objectif

Utilisez cette procédure pour voir les différents types de journaux dans Integra.

Conditions préalables

L’utilisateur doit comprendre l’utilisation des différents types de journaux dans Integra afin de diffuser la transaction en choisissant les options de journal adéquates et en renseignant tous les champs de journal requis dans Integra.

Étapes

  1. Accédez aux modules Grand livre > Configuration du journal > Noms de journal.

  1. Examinez la liste des noms de journal dans l’onglet à gauche.

Résultat

Vous avez examiné une liste complète des noms de journal dans Integra.

Utiliser le connecteur de données dans Excel pour éditer/télécharger les lignes du journal

Le connecteur de données fournit une méthode alternative pour les écritures de journal uniquement lorsqu’il n’y a aucune transaction de projet. Utilisez la toute dernière version du Modèle de journal DynAd (téléchargez la version la plus récente dans le dossier maître de l’IRC dans Box).

Note : cette fonction est uniquement utilisée pour télécharger des journaux n’ayant AUCUNE transaction de projet. Ce processus concerne principalement les comptes de bilan ou les comptes de dépenses/revenus non associés au module Projet.

Étapes

  1. Assurez-vous que les données sont déjà dans Excel avant de continuer.
  2. Accédez aux modules Grand livre > Saisies du journal > Journaux généraux.
  3. Cliquez sur Nouveau après avoir sélectionné le Nom de journal adéquat, p. ex. « JE ».
  4. Cliquez sur l’icône Microsoft (en haut à droite) et sélectionnez Écriture de ligne de journal général.

  1. Dans la fenêtre qui s’affiche, cliquez sur Télécharger.
  2. Ouvrez le fichier téléchargé dans Excel.
  3. Attendez jusqu’à ce qu’un message contextuel s’affiche.

  1. Cliquez sur Oui lorsqu’on vous demande de confirmer le rafraîchissement. Un traitement supplémentaire a lieu, et votre journal est prêt à être renseigné.
  2. Dans la colonne AG, faites passer le curseur sur la ligne bleue, faites-la glisser vers le bas jusqu’à atteindre le nombre de lignes prévu dans le journal. Les deux captures d’écran montrent les modifications de la colonne AG.

  1. Renseignez les Aspects financiers en fonction de la structure comptable de l’IRC.
  2. Une fois le fichier renseigné, cliquez sur Publier.

Le fichier est exécuté et montre les données rafraîchies en bas de l’écran.

  1. Revenez à Integra, accédez à Grand livre > Écritures de journal > Journaux généraux et cliquez sur Montrer uniquement créé par l’utilisateur pour trouver le journal.
  2. Suivez les mêmes étapes pour joindre des documents et faire une soumission au flux de travail que celles indiquées pour les écritures dans le journal DynAd et le Journal direct dans Integra.

Résultat

Le fichier est importé avec succès, et le journal peut être examiné, validé et diffusé.
